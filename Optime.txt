CODE REVIEW AND OPTIMIZATION REPORT (MusB Research Website)

1. CODEBASE ORGANIZATION
-------------------------
- [Backend] Root directory is cluttered with 30+ utility/seed scripts (e.g., seed_*.py, recreate_*.py). 
  * Recommendation: Move all standalone scripts to a /scripts folder.
- [Backend] Log files (server_output.log, traceback.txt) are in the root.
  * Recommendation: Move to a /logs folder and add to .gitignore.
- [Frontend] Project structure is "flat" (no /src directory). 
  * Recommendation: Move all source code (App.tsx, index.tsx, components/, views/) into a /src folder for standard Vite/React organization.
- [Frontend] api.ts contains many endpoints (e.g., news, team, etc.) that point to missing backend routes since the backend models were removed.
  * Recommendation: Clean up orphaned API functions to avoid developer confusion.

2. ERRORS AND BUGS
------------------
- [Frontend] Potential runtime 404s: Components calling fetchHomeSettings, fetchNews, etc. will fail because these routes are not defined in backend/musb_backend/urls.py.
- [Backend] Contact Submission: SubmissionCreateView in contact/views.py uses hardcoded default email "info@musbresearch.com" which might not be synchronized with environment variables.
- [Backend] Missing Error Handling: The Resend email integration logs errors to console but doesn't implement a retry mechanism or robust error reporting for failed email deliveries.

3. SECURITY AND PERFORMANCE
---------------------------
- [Backend] DEBUG mode: Ensure DEBUG is False in production. Currently, it allows CORS from all origins if DEBUG=True.
- [Backend] SECRET_KEY: Has a default value in settings.py. Never use defaults in production.
- [Frontend] Dependency Management: React 19 and Vite 7 are used, which is great, but ensure peer dependencies are met during build.

4. SUGGESTED ACTION PLAN
------------------------
- Step 1: Restructure backend by moving logs and scripts.
- Step 2: Restructure frontend by creating a /src directory.
- Step 3: Remove dead code in frontend/api.ts.
- Step 4: Add logging infrastructure for contact form errors.
